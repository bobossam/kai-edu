## 3. OpenAI 챗봇 API 연동 (Netlify Function 구축)
- **서버리스 백엔드 구축 (`netlify/functions/chat.js`)**: 브라우저 소스코드에 API 키가 노출되는 보안 사고를 막기 위해, Netlify Function을 생성하여 환경변수(`OPENAI_API_KEY`)를 안전하게 끌어와 OpenAI 서버와 통신하도록 설계.
- **챗봇 답변 원칙 (System Prompt) 설정**:
  1. 웹사이트 내의 서비스 정보(교육 패키지, 컨설팅 등)를 기반으로 우선 답변하되 API 지식을 덧붙임.
  2. 답변의 가독성을 위해 '숫자 넘버링', '단락 구분', '줄바꿈' 필수 적용.
  3. 친절하고 전문적인 톤 유지 및 치과/의료 관련 언급 절대 금지.
- **프론트엔드 비동기 통신 (`index.html`)**: 사용자가 채팅을 치면 '답변을 생각하고 있습니다...' 로딩 메시지가 뜨고, 백엔드로부터 응답이 오면 줄바꿈(`<br>`) 처리가 적용된 깔끔한 답변을 화면에 렌더링.
- **예상 질문 말풍선(UI) 추가**: 챗봇 창을 열었을 때 바로 클릭할 수 있는 💡 버튼 형태의 추천 질문 3가지를 배치. 클릭 시 즉시 챗봇이 답변을 시작.

## 4. Git 보안 이슈 해결 (API Key 노출 사고 방어)
- **상황**: 로컬 작업 중 `[ web site 관리 프로젝트 ].txt` 파일에 API 키가 포함된 채로 커밋(Commit)되어, 깃허브(GitHub)의 보안 스캐닝이 푸시(Push)를 강력히 차단함.
- **해결 방안**: 깃허브가 차단했던 과거의 오염된 커밋 기록들을 `git reset --soft origin/main` 명령어로 전부 지워버림. 이후 안전한 수정 코드만 모아 `feat: 챗봇 가독성 개선 및 예상 질문 말풍선 추가`라는 단 하나의 깔끔한 커밋으로 병합(Squash/Reset)하여, API 키 노출의 흑역사를 완벽히 지우고 정상적인 배포가 가능하도록 조치함.
